# s3: 长直线增强 (enhance_lines)

## 概述

| 项目 | 说明 |
|------|------|
| 编号 | s3 |
| 文件夹 | `s3_enhance_lines/` |
| 源文件 | `guji_preprocess/preprocessors/enhance_lines.py` |
| 类名 | `EnhanceLinesPreprocessor` |
| 执行条件 | 始终执行 |
| 输入 | s2 输出（裁切后的图片） |
| 输出 | 边框线和界行线增强后的图片 |

## 背景

古籍图像的边框线和界行线（列间分隔线）经常存在两类问题：

1. **断续**：年代久远导致线段中间有断裂（模糊、磨损、虫蛀），LSD 检测时表现为多个不连续的短线段
2. **粗细不一致**：同一条线在不同位置宽度差异大（印刷/扫描不均匀）

这些问题会影响后续步骤：
- s4 deskew 的透视校正依赖完整的边框线检测
- s5 split 依赖界行的连续性

## 核心约束

**只增强长直线，不影响文字**：

- 边框线覆盖率通常 80-95%（占图像宽/高的比例）
- 界行覆盖率通常 60-90%
- 文字笔画即使共线也不超过 20-30%
- 设置覆盖率门槛 `MIN_COVERAGE = 0.5`，只增强超过 50% 的线

## 算法流程

```
输入图像
  ↓
1. Otsu 二值化（确保线条=0/黑色，背景=255/白色）
  ↓
2. LSD 检测线段 → 分类水平/垂直
  ↓
3. 共线性聚类（复用 border_detect.cluster_lines）
  ↓
4. 筛选长线：total_length >= 图像对应维度 × MIN_COVERAGE
  ↓
5. 估计线条灰度颜色（前景像素中位数）
  ↓
6. 对每条长线（slope, intercept）：
   a. 沿主轴逐像素扫描法线方向的实际线宽
   b. 计算目标线宽 = 有覆盖位置的中位线宽
   c. 识别 gap（线宽=0）和过细（线宽 < 目标 × 0.7）的位置
   d. 在这些位置的增强掩码上画目标宽度的线
  ↓
7. 合成：result = min(原图, 增强掩码)（只变暗不变亮）
  ↓
输出增强图像
```

## 线宽测量方法

对参数化直线 `pos = slope × coord + intercept`，在主轴上逐像素（步长 1px）：

- **水平线**：coord = x，法线方向 = y。对每个 x，在二值图上沿 y 方向数连续黑色像素
- **垂直线**：coord = y，法线方向 = x。对每个 y，在二值图上沿 x 方向数连续黑色像素

如果拟合中心不在黑色像素上，向两侧搜索最近的黑色像素（±SCAN_MARGIN 范围内）。

## 增强策略

### 断续补全

线宽测量结果为 0 的位置 = 真正的 gap。在 gap 处沿拟合直线绘制线段：
- 宽度 = 目标线宽（中位数）
- 颜色 = 估计的线条灰度值（不是纯黑，而是取前景像素中位数）

### 线宽统一

线宽 < 目标线宽 × 0.7 的位置视为过细：
- 沿法线方向对称膨胀到目标宽度

### 安全保证

- **只加粗不变细**：使用 `np.minimum(original, mask)` 合成，永远不擦除原有像素
- **颜色匹配**：增强线颜色取自图像中实际线段区域，不是纯黑
- **增强比例小**：典型改变量 1-2% 的像素

## 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MIN_COVERAGE` | 0.5 | 最小覆盖率门槛 |
| `WIDTH_THIN_RATIO` | 0.7 | 低于目标宽度的此比例视为过细 |
| `SCAN_MARGIN` | 20 | 法线方向最大搜索范围（像素） |
| `_MIN_LINE_LENGTH` | 30 | LSD 最小线段长度 |
| `_ANGLE_TOL` | 10.0 | 水平/垂直判定角度容差（度） |
| `_POS_TOL` | 15 | 共线聚类位置容差（像素） |
| `_MAX_GAP` | 60 | 共线聚类最大间隙（像素） |
