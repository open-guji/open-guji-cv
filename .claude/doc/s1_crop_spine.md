# s1: 裁书脊阴影 (crop_spine)

## 概述

| 项目 | 说明 |
|------|------|
| 编号 | s1 |
| 文件夹 | `s1_crop_spine/` |
| 源文件 | `guji_preprocess/preprocessors/crop_spine.py` |
| 类名 | `CropSpinePreprocessor` |
| 执行条件 | `profile.has_spine_shadow == True` |
| 输入 | 原始图片（可能有页边距） |
| 输出 | 裁掉书脊阴影侧后的图片 |

## 背景

部分古籍扫描图片在装订侧（左侧或右侧）会出现**书脊阴影**——一条从上到下贯穿的暗色纵向条纹。例如 book4 有明显的书脊阴影。如果不裁掉，会干扰后续的边框检测和二值化。

## 设计要点

### 执行顺序

s1 在 s2（裁边框）**之前**执行，此时图片还有页边距。如果直接在含页边距的图上检测书脊，黑色填充背景（如 book4）可能被误认为书脊。

**解决方案**：s1 内部先调用 `find_content_bounds()` 定位内容区域，然后仅在内容区域内检测书脊。

### 检测算法

```
1. find_content_bounds(gray) → 定位内容区域
2. 在内容区左侧 15% 和右侧 15% 分别检测：
   a. 取该条带的逐列平均亮度
   b. 找最暗列（书脊中心）
   c. 与远端亮度对比，差值 > DARKNESS_THRESHOLD(20) 视为有阴影
   d. 找到阴影结束位置（亮度恢复到 60% 水平）
3. 比较左右两侧置信度，裁掉分数更高的一侧
4. 置信度阈值 0.3，低于此则不裁切
```

### 裁切坐标

裁切在**原图坐标系**中操作，而不是内容区坐标系。确保页边距（如果后续 s2 需要裁的话）仍然保留在非书脊侧。

## 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `EDGE_SEARCH_RATIO` | 0.15 | 搜索书脊的边缘区域宽度（相对于内容区宽度） |
| `DARKNESS_THRESHOLD` | 20 | 暗度差值阈值（灰度值） |
