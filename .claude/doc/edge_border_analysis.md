# 边缘竖边框检测分析（2026-02-21）

## 目标

通过检测内容区左右边缘是否有竖边框线，作为判断 cut_half / uncut_full 的辅助信号。

## 核心思路

用户提出的区分特征：
- **已剪切 (cut_half)**：一侧是裁切边（横向边框延伸到最边上，或竖边框紧贴边缘），另一侧有空白
- **未剪切 (uncut_full)**：左右两边都有空白，边框不到边

## 三种检测方法及结果

### 方法1: 最外侧列的最长连续暗线段（固定阈值128）

检测内容区最外侧 15 列中，灰度 < 128 的最长连续暗像素段占图像高度的比例。

| 书 | 结果 | 有效性 |
|---|---|---|
| book2 (uncut) | 两侧 BORDER (run=0.77-0.83) | ✅ 正确 |
| book4 (cut) | 单侧 BORDER=1.0（黑色填充侧）| ✅ 正确 |
| book1/3/5 (cut) | 两侧都无 BORDER (<0.58) | ❌ 不确定 |

**book3 失败原因**：彩色底灰度 161-197，边框灰度 172-177，固定阈值 128 远低于边框亮度。

### 方法2: Sobel 竖边缘 + 距内容区边缘距离

用 Sobel X 检测竖边缘，找第一条强竖边缘距内容区边缘的距离（offset）。
判断：offset < 2% → AT_EDGE（边框贴边），> 2% → inside（边框在内部有边距）。

| 书 | 左侧 | 右侧 | 区分度 |
|---|---|---|---|
| book1 (cut) | 0.1-1.3% AT_EDGE | 0.1-1.4% AT_EDGE | ❌ 两侧都在边缘 |
| book2 (uncut) | 0.1-0.2% AT_EDGE | 0.1-0.4% AT_EDGE | ❌ 两侧都在边缘 |
| book3 (cut) | 0.0-1.4% AT_EDGE | 0.0-1.2% AT_EDGE | ❌ 两侧都在边缘 |
| book4 (cut) | **交替 0.1%/3.7%** | **交替 3.7%/0.1%** | ✅ 明显不对称 |
| book5 (cut) | 0.1-0.5% AT_EDGE | 0.2-0.5% AT_EDGE | ❌ 两侧都在边缘 |

**问题**：白色扫描背景去除后，内容区边缘本身就产生 Sobel 响应，无法区分"边框线"和"内容边界过渡"。

### 方法3: 分段分析（book3 详细）

对 book3/1.png 用 Sobel 找到左侧外边框在 col=134（距左边缘 5.2%）：
- 段0 (顶部 0-20%): 边框完全不存在
- 段1-4 (20-100%): 边框断续出现

边框不贯穿到顶部，与"已剪切页面横框不延伸到最上方"的特征一致，但信号太弱且不稳定。

## 各书详细边框位置（Sobel 检测）

### book3/1.png 左侧列亮度分布
```
col   0~ 24: 161 → 178 (纸张底色渐变, 无边框)
col  25~ 50: 179 → 197 (继续渐变)
col 128~144: 206→192→184→177→173→173→177→184→192→197→202→204→207→208 (col 134-136 是暗沟谷=边框)
```
外边框在 col=134, 距边缘 5.2%, 灰度 177 (底色 ~200, 落差仅 ~23 灰度级)

### book4 不对称模式（交替出现）
```
1.png:  L: AT_EDGE(0.1%)  R: inside(3.7%)  ← 奇数页=右半页, 左侧是裁切边
10.png: L: inside(3.7%)   R: AT_EDGE(0.1%) ← 偶数页=左半页, 右侧是裁切边
```
交替模式本身就是 cut_half 的强证据。offset=3.7% (120px) 来自黑色填充区。

## 结论

| 检测模式 | 判定 | 置信度 | 出现在 |
|---------|------|--------|-------|
| 两侧都有完整竖线 (run>0.6) | uncut_full | 高 | book2 |
| 单侧有完整竖线，另一侧无 | cut_half | 高 | book4 |
| 两侧都无完整竖线 | 不确定 | — | book1/3/5（回退到宽高比）|

**核心限制**：
1. 有白色扫描背景的书：去背景后内容区边缘自带边缘响应，掩盖边框信号
2. 彩色底书：边框与底色的灰度差太小（~23），固定阈值失效
3. 磨损严重的边框：连续性被破坏，无法形成贯穿竖线

**后续方向**：先处理页边距裁切（去除白色扫描背景），使内容区更贴近实际古籍页面，再重新评估此信号。
