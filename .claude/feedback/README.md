# OCR 质量反馈

> 来源：guji-platform/merger 模块
> 用途：根据下游使用情况反馈 OCR 质量问题

## 目录说明

此目录接收来自 `guji-platform` 项目的 OCR 质量反馈。

**文件命名**：`merger_quality_YYYYMMDD.md`

**更新频率**：每次 `guji-platform` 运行质量分析时自动同步

## 如何使用

1. **查看最新反馈**：
   ```bash
   # 找到最新日期的文件
   ls -lt *.md | head -1

   # 查看内容
   cat merger_quality_20260226.md
   ```

2. **理解反馈内容**：
   - **整体质量汇总表**：各册的 Title 匹配率、Detail 使用率
   - **需要优先改进的册**：质量最低的册
   - **具体改进建议**：针对性的优化方向

3. **更新 todo.md**：
   ```bash
   # 根据反馈添加任务到 .claude/todo.md
   # 例如：
   # - [ ] 修复 ce01 最后12本书的简繁混杂问题（Title 92.8% → 99%+）
   # - [ ] 增强夹注检测覆盖率（Detail 97.7% → 99%+）
   ```

4. **优化代码**：
   - 修改 `src/detectors/char_grid.py`（夹注检测）
   - 修改 `border_detect.py`（版面检测）
   - 修改 `ocr_postprocess.py`（OCR 后处理，如需要）

5. **重新生成 OCR**：
   ```bash
   python3 src/run_ocr.py --book book1 --volume 1
   ```

6. **提交改动**：
   ```bash
   git commit -m "OCR优化：根据 merger 反馈修复XXX问题"
   ```

## 质量指标说明

### Title 匹配率

**定义**：merger 能够成功匹配的书名数量 / 总书目数量

**目标**：**99%+**

**影响因素**：
- OCR 书名识别准确性（简繁、符号）
- 书名列版面检测质量
- 书名列 OCR 文本清洗

**< 95% 说明有问题**：需要检查书名相关的 OCR 流程

### Detail 使用率

**定义**：成功填充的 detail 字符数 / 总 detail 字符数

**目标**：**99%+**

**影响因素**：
- 夹注列检测覆盖率
- 版面检测完整性（是否有整页丢失）
- OCR 字符网格质量（text=None 数量）

**< 98% 说明有问题**：需要检查夹注检测和版面检测

### 行数差距

**定义**：自动生成 TeX 行数 - 参考 TeX 行数

**目标**：**< 5 行**

**影响因素**：
- 版面检测失败的页面
- 夹注列漏检
- 特殊内容（总结行、附录）处理

**> 10 行说明有问题**：需要排查版面/夹注检测

- `易稽览圆` ← 应为 `易緯稽覽圖`

**建议**：增强 OCR 后处理，繁简转换

**优化方向**：
1. 创建 `src/ocr_postprocess.py` 模块
2. 实现繁简转换函数（使用 OpenCC 或字典）
3. 在 char_grid.py 的 OCR 文本提取后调用

**预期效果**：Title 匹配率 92.8% → 99%+
